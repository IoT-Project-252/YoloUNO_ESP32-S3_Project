\subsection{Task 3 - Theo dõi nhiệt độ và độ ẩm với màn hình LCD}

Task 3 tập trung vào ba yêu cầu cốt lõi của đề bài: (1) đo nhiệt độ/độ ẩm theo chu kỳ, (2) hiển thị ít nhất 3 trạng thái trên LCD dựa theo ngưỡng đo, và (3) loại bỏ các biến toàn cục rời rạc bằng mô hình dữ liệu dùng chung có đồng bộ trong FreeRTOS.

\subsubsection*{Mục tiêu hiện thực}

Nhóm hiện thực Task 3 theo hướng tách rõ chức năng thành hai task FreeRTOS độc lập:
\begin{itemize}
    \item \texttt{temp\_humi}: đọc DHT20 và cập nhật dữ liệu đo vào vùng dùng chung.
    \item \texttt{displayLCD}: đọc trạng thái từ semaphore và render nội dung LCD tương ứng.
\end{itemize}

Hai task này được tạo trong \texttt{setup()} của \texttt{main.cpp}:
\begin{lstlisting}[language=c, caption=Khởi tạo các task liên quan Task 3 trong main.cpp]
xTaskCreate(temp_humi,  "TempHumi",   4096, projectSharedData, 1, NULL);
xTaskCreate(displayLCD, "DisplayLCD", 4096, projectSharedData, 1, NULL);
\end{lstlisting}

\subsubsection*{Tổ chức dữ liệu chia sẻ và loại bỏ global variables}

Thay vì dùng nhiều biến toàn cục rời rạc, hệ thống gom toàn bộ trạng thái dùng chung vào \texttt{SharedData}. Với Task 3, các thành phần quan trọng gồm:
\begin{itemize}
    \item \texttt{temperature}, \texttt{humidity}: dữ liệu đo mới nhất.
    \item \texttt{mutex}: bảo vệ truy cập dữ liệu đo (tránh race condition).
    \item \texttt{i2cMutex}: bảo vệ bus I2C dùng chung giữa DHT20 và LCD.
    \item \texttt{semNormal}, \texttt{semWarning}, \texttt{semCritical}: tín hiệu trạng thái hiển thị.
\end{itemize}

\begin{lstlisting}[language=c, caption=Cấu trúc SharedData dùng cho Task 3 (trích global.h)]
struct SharedData
{
    float temperature;
    float humidity;

    SemaphoreHandle_t mutex;
    SemaphoreHandle_t i2cMutex;

    SemaphoreHandle_t semNormal;
    SemaphoreHandle_t semWarning;
    SemaphoreHandle_t semCritical;
};
\end{lstlisting}

Cách tổ chức này đảm bảo dữ liệu được quản lý tập trung, giảm phụ thuộc chéo giữa module và phù hợp yêu cầu ``remove ALL global variables'' theo tinh thần đề bài.

\subsubsection*{Task đọc cảm biến DHT20 và phát tín hiệu trạng thái}

Task \texttt{temp\_humi} thực hiện ba bước chính trong mỗi chu kỳ:
\begin{enumerate}
    \item Khóa \texttt{i2cMutex} để đọc DHT20 an toàn.
    \item Khóa \texttt{mutex} để ghi \texttt{temperature/humidity} vào vùng chia sẻ.
    \item So sánh ngưỡng nhiệt độ và \texttt{xSemaphoreGive()} semaphore trạng thái tương ứng.
\end{enumerate}

Đoạn logic ngưỡng:
\begin{lstlisting}[language=c, caption=Phát semaphore theo ngưỡng nhiệt độ (trích temp\_humi.cpp)]
if (t >= 20 && t < 30)
{
    xSemaphoreGive(sharedData->semNormal);
}
else if ((t >= 30.0 && t < 38.0) || (t >= 7 && t < 20))
{
    xSemaphoreGive(sharedData->semWarning);
}
else
{
    xSemaphoreGive(sharedData->semCritical);
}
\end{lstlisting}

Tương ứng, hệ thống có ba mức trạng thái hiển thị như Bảng~\ref{tab:task3-thresholds}.

\begin{table}[h!]
\centering
\caption{Ngưỡng trạng thái hiển thị của Task 3}
\label{tab:task3-thresholds}
\begin{tabular}{|c|c|c|}
\hline
\textbf{Khoảng nhiệt độ} & \textbf{Trạng thái} & \textbf{Semaphore phát} \\
\hline
$20 \leq T < 30$ & NORMAL & \texttt{semNormal} \\
\hline
$30 \leq T < 38$ hoặc $7 \leq T < 20$ & WARNING & \texttt{semWarning} \\
\hline

\end{table}

\subsubsection*{Task hiển thị LCD theo mô hình state machine}

Task \texttt{displayLCD} hoạt động như một state machine dựa trên semaphore. Ở mỗi vòng lặp:
\begin{itemize}
    \item Task kiểm tra lần lượt \texttt{semNormal}, \texttt{semWarning}, \texttt{semCritical} theo cơ chế non-blocking.
    \item Khi nhận được semaphore, task khóa \texttt{mutex} để lấy bản sao dữ liệu đo.
    \item Task khóa \texttt{i2cMutex} để render nội dung lên LCD, tránh tranh chấp bus I2C với task cảm biến.
\end{itemize}

\begin{lstlisting}[language=c, caption=Khối xử lý trạng thái WARNING trên LCD (trích display\_lcd.cpp)]
else if (xSemaphoreTake(sharedData->semWarning, 0) == pdTRUE)
{
    if (xSemaphoreTake(sharedData->mutex, pdMS_TO_TICKS(50)) == pdTRUE)
    {
        localTemp = sharedData->temperature;
        localHum = sharedData->humidity;
        xSemaphoreGive(sharedData->mutex);
    }

    if (xSemaphoreTake(sharedData->i2cMutex, pdMS_TO_TICKS(100)) == pdTRUE)
    {
        lcd.clear();
        lcd.setCursor(0, 0); lcd.print("STATE: WARNING!");
        lcd.setCursor(0, 1); lcd.print("T:"); lcd.print(localTemp, 1);
        lcd.print("C H:"); lcd.print(localHum, 0); lcd.print("%");
        xSemaphoreGive(sharedData->i2cMutex);
    }
}
\end{lstlisting}

Riêng trạng thái \texttt{CRITICAL}, nhóm bổ sung hiệu ứng nhấp nháy đèn nền LCD (backlight blink) để tăng khả năng cảnh báo trực quan cho người dùng.

\subsubsection*{Điểm nhấn kỹ thuật của Task 3}

\begin{itemize}
    \item \textbf{Đồng bộ đa tầng}: dùng cả mutex dữ liệu và mutex I2C để xử lý đồng thời hai loại xung đột khác nhau.
    \item \textbf{Tách producer-consumer}: task cảm biến chỉ phát tín hiệu trạng thái, task LCD chịu trách nhiệm hiển thị.
    \item \textbf{Thiết kế mở rộng}: cấu trúc Task 3 có thể tái sử dụng cho Task 4 (web dashboard) và Task 6 (publish cloud) nhờ vùng dữ liệu chia sẻ chuẩn hóa.
    \item \textbf{Ổn định thời gian chạy}: các nhánh lỗi (đọc DHT20 thất bại hoặc không lấy được mutex) đều có timeout và retry, giúp tránh treo hệ thống.
\end{itemize}

\subsubsection*{Tổng kết Task 3}

Task 3 đã đáp ứng các yêu cầu bắt buộc của đề bài: có cơ chế phát semaphore theo điều kiện đo, có ít nhất 3 trạng thái hiển thị trên LCD, và dùng mô hình dữ liệu chia sẻ có bảo vệ mutex thay cho cách dùng biến toàn cục rời rạc. Kết quả là hệ thống chạy ổn định trong môi trường đa task và tạo nền tảng cho các task mở rộng ở các chương tiếp theo.

\subsection{Task 4 - Webserver ở chế độ Access Point}

\subsubsection*{Mục tiêu và phạm vi}

Task 4 được triển khai nhằm đáp ứng các yêu cầu của đề bài về giao diện web điều khiển thiết bị trong chế độ Access Point. Nhóm hiện thực theo kiến trúc:
\begin{itemize}
    \item ESP32-S3 tự phát WiFi AP để người dùng truy cập trực tiếp.
    \item Dashboard hiển thị dữ liệu nhiệt độ/độ ẩm realtime.
    \item Dashboard có hai cụm điều khiển: NeoPixel RGB và quạt PWM.
    \item Trang \texttt{/settings} cho phép cấu hình STA để chuyển sang chế độ AP+STA.
\end{itemize}

\subsubsection*{Kiến trúc task và mô hình dữ liệu}

Về mặt thiết kế, Task 4 gồm hai task chính:
\begin{itemize}
    \item \texttt{actuatorTask}: điều khiển phần cứng NeoPixel và quạt theo trạng thái trong \texttt{SharedData}.
    \item \texttt{webServerTask}: chạy web server, xử lý route HTTP và cập nhật trạng thái điều khiển.
\end{itemize}

\begin{lstlisting}[language=c, caption=Khối tạo task Task 4 trong setup (trích main.cpp)]
// xTaskCreate(actuatorTask,   "Actuators",   4096, projectSharedData, 1, NULL);
xTaskCreate(webServerTask,  "WebServer",   8192, projectSharedData, 1, NULL);
\end{lstlisting}

Trong phiên bản hiện tại phục vụ kiểm thử MQTT mở rộng, nhóm đang giữ \texttt{webServerTask} hoạt động và tạm comment một số task khác (bao gồm \texttt{actuatorTask}) để cô lập luồng test. Khi demo đầy đủ Task 4, chỉ cần bật lại các lệnh \texttt{xTaskCreate(...)} tương ứng.

Để đảm bảo thread-safety khi web callback và task phần cứng truy cập cùng dữ liệu, nhóm dùng:
\begin{itemize}
    \item \texttt{mutex}: bảo vệ dữ liệu cảm biến \texttt{temperature/humidity}.
    \item \texttt{actuatorMutex}: bảo vệ trạng thái LED/quạt (\texttt{ledOn, ledR, ledG, ledB, fanOn, fanSpeed}).
\end{itemize}

\subsubsection*{Khởi tạo AP và chuyển đổi AP+STA}

Khi khởi động, \texttt{webServerTask} đưa WiFi về chế độ AP và phát mạng cục bộ:
\begin{lstlisting}[language=c, caption=Khởi tạo chế độ Access Point (trích web\_server.cpp)]
WiFi.mode(WIFI_AP);
WiFi.softAP(SSID_AP, PASS_AP);
Serial.println("[WebServer] AP started.");
Serial.print("[WebServer] AP IP: ");
Serial.println(WiFi.softAPIP());
\end{lstlisting}

Khi người dùng gửi SSID/password từ trang \texttt{/settings}, API \texttt{/api/wifi} sẽ:
\begin{enumerate}
    \item chuyển sang \texttt{WIFI\_AP\_STA},
    \item thử \texttt{WiFi.begin()},
    \item cập nhật \texttt{staConnected} và \texttt{staSSID} nếu thành công,
    \item nếu thất bại thì quay về \texttt{WIFI\_AP}.
\end{enumerate}

\subsection{Task mở rộng - Giao tiếp 2 thiết bị qua Local MQTT Broker}

\subsubsection*{Mục tiêu mở rộng}

Nhóm mở rộng hệ thống theo hướng giao tiếp hai vi điều khiển qua local MQTT broker (Mosquitto):
\begin{itemize}
    \item Thiết bị A (YoloUNO ESP32-S3): đọc cảm biến PIR và publish sự kiện chuyển động.
    \item Thiết bị B (ESP32-S3 DevKitC-1): subscribe sự kiện, kích hoạt buzzer 5V cảnh báo.
\end{itemize}

Mục tiêu của phần mở rộng là chứng minh khả năng liên động liên thiết bị theo mô hình publish/subscribe trong cùng mạng LAN.

\subsubsection*{Kiến trúc tổng thể và broker cục bộ}

Mosquitto được chạy trên máy tính trong mạng nội bộ, mở cổng 1883. Cả hai ESP32 kết nối broker cùng địa chỉ IP LAN (ví dụ \texttt{192.168.1.5}).

Luồng dữ liệu:
\begin{enumerate}
    \item Device A phát hiện chuyển động từ PIR.
    \item Device A publish JSON lên topic sự kiện.
    \item Device B subscribe topic này, nhận bản tin và kích hoạt buzzer.
    \item Device B publish ACK trạng thái buzzer để Device A/log theo dõi.
\end{enumerate}

\subsubsection*{Thiết kế topic và payload MQTT}

doc["device_id"] = MQTT_CLIENT_ID_A;

\subsubsection*{Device A: Task PIR + MQTT publisher}

Trên project chính, nhóm tạo \texttt{pirMqttTask} với các cơ chế:
\begin{itemize}
    \item Chờ STA sẵn sàng (\texttt{staConnected}) trước khi kết nối MQTT.
    \item Tự reconnect MQTT mỗi 5 giây khi mất kết nối.
    \item Đọc PIR theo chu kỳ 200ms, phát hiện cạnh lên (LOW $\rightarrow$ HIGH).
    \item Dùng cooldown để chống spam bản tin liên tiếp.
\end{itemize}

\begin{lstlisting}[language=c, caption=Phát hiện cạnh lên PIR và kiểm soát cooldown (trích pir\_mqtt.cpp)]
bool currentPir = digitalRead(PIR_PIN) == HIGH;
if (currentPir && !lastPirState) {
    uint32_t now = millis();
    if (now - lastPublishTime >= PIR_COOLDOWN_MS) {
        publishMotionEvent(true);
        lastPublishTime = now;
    }
}
lastPirState = currentPir;
\end{lstlisting}

\subsubsection*{Device B: MQTT subscriber + buzzer driver}

Trên project \texttt{ESP32-S3-DevKit-C1\_Src}, nhóm chia thành ba task:
\begin{itemize}
    \item \texttt{mqttTask}: quản lý kết nối WiFi/MQTT và xử lý callback.
    \item \texttt{buzzerTask}: bật/tắt buzzer theo cờ trigger, publish ACK.
    \item \texttt{blinkLedTask}: nháy LED onboard để báo board đang hoạt động.
\end{itemize}

Chiến lược xử lý event chồng lấp: nếu buzzer đang kêu thì bỏ qua trigger mới để tránh cảnh báo kéo dài vô hạn.

\begin{lstlisting}[language=c, caption=Logic bỏ qua sự kiện mới khi buzzer đang active (trích Device B main.cpp)]
if (motion && strcmp(event, "motion_detected") == 0) {
    if (xSemaphoreTake(alarmMutex, pdMS_TO_TICKS(50)) == pdTRUE) {
        if (alarmActive) {
            Serial.println("[Buzzer] Alarm already active. Ignoring new event.");
        } else {
            alarmTrigger = true;
        }
        xSemaphoreGive(alarmMutex);
    }
}
\end{lstlisting}

\subsubsection*{Kết nối phần cứng của phần mở rộng}

\textbf{PIR (Device A):}
\begin{itemize}
    \item Chân tín hiệu PIR nối vào GPIO cấu hình trong \texttt{PIR\_PIN} (mã hiện tại dùng GPIO8).
    \item Cấu hình \texttt{INPUT\_PULLDOWN} để tránh trạng thái floating.
\end{itemize}

\textbf{Buzzer 5V (Device B):}
\begin{itemize}
    \item Không nối trực tiếp buzzer 5V vào GPIO ESP32.
    \item Dùng transistor/MOSFET driver, nối mass chung giữa nguồn 5V và ESP32.
    \item GPIO chỉ đóng vai trò tín hiệu điều khiển transistor.
\end{itemize}

\subsubsection*{Quy trình kiểm thử end-to-end}

\begin{enumerate}
    \item Chạy Mosquitto local broker và xác nhận cổng 1883 hoạt động.
    \item Nạp firmware cho Device A và Device B.
    \item Mở serial monitor cả hai board để quan sát trạng thái WiFi/MQTT.
    \item Tạo chuyển động trước PIR.
    \item Kiểm tra Device A publish event, Device B nhận event và phát còi.
    \item Kiểm tra bản tin ACK của buzzer được publish ngược lại.
\end{enumerate}

\subsubsection*{Giá trị của phần mở rộng}

Tính năng mở rộng này giúp hệ thống vượt khỏi mô hình đơn thiết bị, thể hiện năng lực:
\begin{itemize}
    \item giao tiếp liên node theo chuẩn MQTT,
    \item triển khai kiến trúc event-driven trong LAN,
    \item và thiết kế hệ cảnh báo tách rời cảm biến/chấp hành để dễ mở rộng thành mạng IoT nhiều node.
\end{itemize}
